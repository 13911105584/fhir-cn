## <span class="sectioncount">1.13.2.0.2<a name="1.13.2.0.2"> </a></span> Interpretation of ElementDefinition in different contexts

The pseudo-data type `ElementDefinition` is used both in [Profile](profile.html) and [ExtensionDefinition](extensiondefinition.html), and even within  those resources, the way its elements are to be used and interpreted depend on the context where `ElementDefinition` used.

### <span class="sectioncount">1.13.2.0.2.1<a name="1.13.2.0.2.1"> </a></span> First (root) element in a Structure

*   Must be a &quot;complex&quot; element (see below)
*   `path` is equal to the name of the type indicated in Structure.type
*   `name` associates a name   <!-- What does this mean? -->
*   <!-- LM: Do this as an invariant -->Prohibited: `slicing`, `nameReference`, `fixed`, `mustSupport`, `isSummary`
*   `type` is set to &quot;DomainResource&quot; or (rarely) &quot;Resource&quot; for constraints on Resources, &quot;Element&quot; for constraints on data types
*   <!-- LM: How is this different for root? -->`pattern` indicates a value that an instance of a while complex type constrained by this structure must match. May be used instead of applying constraints to the nested elements of the data type.
*   <!-- LM: How is this different for root? -->`example` example value for a complex type in an instance that complies to the structure
*   <!-- LM: Do this as an invariant? -->`min` and `max` are prohibited unless defining an extension, in which case they set outer bounds on how the element can appear when referenced in a profile
*   `isModifier` is prohibited when defining resources or data types.  When defining extensions, this indicates whether the extension is a modifier extension
*   <!-- Handle first part as invariant -->`isSummary` is prohibited when defining resources or data types.  When defining extensions, this indicates whether the extension is a modifier extension

### <span class="sectioncount">1.13.2.0.2.2<a name="1.13.2.0.2.2"> </a></span> Nested element in a structure
(i.e. not root)

*   `slicing` can only be used on the first repeat of an element, this element is considered the slicing entry

### <span class="sectioncount">1.13.2.0.2.3<a name="1.13.2.0.2.3"> </a></span> Element in a structure acting as the slicing entry

The `ElementDefinition` is understood to be the set of constraints to be used for the &quot;open&quot; slice, i.e. when the slice is open, this definition is used as a constraint on elements in the instance that are not part of a slice and belong to the open portion of the slice. Its use follows the &quot;normal case&quot;, except:

*   `slicing` must be present
*   `min` and `max` govern the number of total occurrences of the sliced element including the number of occurrences in the open portion of the slice.

### <span class="sectioncount">1.13.2.0.2.4<a name="1.13.2.0.2.4"> </a></span> Resource and Extension definitions

*   Prohibited: `name`
*   <!-- Spreadsheet doesn't support this, but seems reasonable/useful -->`slicing` is only supported when slicing based on type (to assert different rules/guidance for resources with multiple types, e.g. &quot;code|integer&quot; or &quot;Reference(Foo|Bar)&quot;

### <span class="sectioncount">1.13.2.0.2.5<a name="1.13.2.0.2.5"> </a></span> Only element in a simple extension

*   `path` is equal to Extension
*   <!-- Don't think mustSupport should exist when defining things -->`mustSupport` may be used, but if it is not present it defaults to &quot;false&quot;

### <span class="sectioncount">1.13.2.0.2.6<a name="1.13.2.0.2.6"> </a></span> First (root) element in a complex extension

*   `path` is equal to Extension
*   `type` is set to `Extension`

### <span class="sectioncount">1.13.2.0.2.7<a name="1.13.2.0.2.7"> </a></span> Nested element in a complex extension

*   `representation` may not be used
*   `slicing` may be used to express constraints on the use of specific types when the element allows a choice of type (name ends in [x])
   <!-- don't understand this use case. just define a single type -->*   `mustSupport` may be used, but if it is not present it defaults to &quot;false&quot;
*   `isModifier` may be used, but if it is not present it defaults to &quot;false&quot;

### <span class="sectioncount">1.13.2.0.2.8<a name="1.13.2.0.2.8"> </a></span> Complex elements (elements with upper-case data type)

*   `representation` is prohibited

  <!-- only be agreement with both ITS and MnM and FHIR core -->*   `maxLength` may not be used
